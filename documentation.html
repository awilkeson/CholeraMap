Beginning Design Process

When I first began thinking about how I was going to place the map and other information on the web page I knew I was going to have a menu at the top followed by a heading explaining what the viewer was looking at. I planned to have the map below. I originally had the deaths per day chart below the map and shown horizontially. I realized that it would be eaiser if the user could see the day chart as they were looking at the map so they could select a day while viewing the map.

In regards to the map design I began by loading all the death datapoints and pumps as cirles on the lines. At first I was only cycling through the first two lines in the json file so not all of the lines of the map were showing. Once I realized this, I fixed it, which provided a more complete map. I originally had the death dots colored red and the pumps blue, but it was difficult to see the pumps among the red dots. I changed the red death does to gray which helped, but realized that a change in shape might make things easier. I changed the pumps to be crosses instead of circles which helped separate them more.

For the sections of filters (Age Range, Gender, and Clusters) I began with age ranges and had them as buttons but discovered you couldn't select more than one at a time and that the viewer may want to see two age groups at a time so they could compare them. I changed them to checkboxes and did the same for the gender selection.

Overall, how I planned my time working on the project and how I progressed through the project. I followed the bulleted lists on the project assignment that listed what needed to be completed to recieve a 'C', 'B', and 'A'. I worked my way from C to A working through the tasks in each one building on the previous So I completed things needed for a C before moving to items needed for a B, etc.

Thoughts throughout Development

Quite a few things were adjusted and changed as I was coding the map and corresponding filters.

Colors

Throughout the process I used high contrasting colors so that I would be able to tell if something was working. Towards the end when I had all the technical components working is when I changed the colors to be more cohesive and allow accesibility by using a color blindness checker.

Coding

As I worked my way through one task such as figuring out how to cluster the data and allow the user to select the size of the cluster and then clearing the clusters from the map helped when I went to have the user select a range of days.

I realized that a lot of things once you figured out how it worked and how to get it to work properly could be applied to other sections throughout the code.

Takeaways

Discovered in Map

As I was coding and testing features I came across a number of things that the original John Snow map or just a glance at the data would be difficult to learn. I've listed them below.

  • There are almost an equal number of Male and Female deaths
  • The two age ranges with the most deaths were 'o-10' and '> 80', which make sense since they would be the youngest and oldest of the population. I think this is also interesting as we recently saw this same pattern with the recent COVID-19 pandemic.
  • Most of the deaths were centered around the pump central to town and near the Brewery
  • There were few deaths at the beginning but they grew exponentially from the 8th of August to September 2nd when before they started to decrease and taper out again
